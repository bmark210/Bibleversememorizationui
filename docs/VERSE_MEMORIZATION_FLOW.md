# Механика работы со стихом в проекте (текущее состояние)

Документ описывает реальный текущий флоу по коду: от добавления стиха до перехода в `MASTERED`, включая логику тренировки, повторений, daily goal и серверные правила сохранения прогресса.

## 1. Модель данных и статусы

### 1.1 База данных (базовый статус)
- В `prisma/schema.prisma` у `UserVerse` хранится базовый статус: `MY`, `LEARNING`, `STOPPED`.
- Также хранятся поля прогресса:
  - `masteryLevel` (int)
  - `repetitions` (int)
  - `lastTrainingModeId` (int | null)
  - `lastReviewedAt` (Date | null)
  - `nextReviewAt` (Date | null)

### 1.2 Отображаемый статус (вычисляемый)
- На фронте и в DTO используются display-статусы: `CATALOG`, `MY`, `LEARNING`, `REVIEW`, `MASTERED`.
- Вычисление:
  - `MASTERED`, если `repetitions >= 5`
  - `REVIEW`, если `masteryLevel >= 7` и не `MASTERED`
  - `LEARNING`, если базовый статус `LEARNING`, но mastery еще ниже порога review
  - `MY`, если базовый статус `MY`
  - `STOPPED`, если базовый статус `STOPPED`

## 2. Полный пользовательский путь стиха

## 2.1 Добавление стиха
- В `AddVerseDialog` пользователь ищет стих через Bolls или выбирает вручную.
- При подтверждении вызывается `POST /api/users/{telegramId}/verses` (в текущей реализации этот endpoint создает/апдейтит глобальный `Verse` в каталоге).
- После этого стих появляется в каталоге (`/api/verses`) как `CATALOG`, пока у пользователя не создана запись `UserVerse`.

## 2.2 Перевод стиха в личный прогресс
- В каталоге при действии `CATALOG -> MY` сначала создается `UserVerse` через `PUT /api/users/{telegramId}/verses/{externalVerseId}`.
- Затем патчится статус (`PATCH`) и стих становится личным (`MY`), затем может перейти в `LEARNING`.

## 2.3 Переходы статусов в UI
- Основные переходы:
  - `CATALOG -> MY`
  - `MY -> LEARNING`
  - `LEARNING/REVIEW/MASTERED -> STOPPED`
  - `STOPPED -> LEARNING`
- Важно: `REVIEW` и `MASTERED` в БД не пишутся как базовые статусы, это display-слой над `LEARNING` + прогресс.

## 3. Планирование дневной тренировки (daily goal)

- Пользователь задает план: сколько `newVersesCount` (изучение) и `reviewVersesCount` (повторение).
- Фронт строит локальный план и хранит сессию дня в `localStorage`.
- Серверный readiness (`GET /api/users/{telegramId}/daily-goal/readiness`) пересчитывает доступность:
  - `learning` доступно, если статус display `LEARNING`
  - `review` доступно только для display `REVIEW` и только если стих `due` (`nextReviewAt <= now` или отсутствует)
- Если `review` запрошен, но доступных due-стихов нет, этап review помечается как skipped.

## 4. Основной боевой флоу тренировки: `VerseGallery`

Главный тренинг сейчас идет через `VerseGallery` + `useTrainingFlow`.

## 4.1 Старт тренировки
- Пользователь открывает карточку стиха и нажимает `Учить`/`Повторять`.
- Если статус `MY` или `STOPPED`, перед стартом делается перевод в `LEARNING`.
- Загружается список стихов пользователя (`LEARNING` + `REVIEW`).
- Для режима тренировок есть фильтры: `catalog`, `learning`, `review`.

## 4.2 Критерий допуска к тренировке
- `LEARNING` стих — допускается всегда.
- `REVIEW` стих — допускается только если наступил срок повторения (`due`).

## 4.3 Выбор режима упражнения
- Режим выбирается через `chooseTrainingModeId(...)`:
  - Основан на `masteryLevel` и `lastTrainingModeId`
  - Для review-stage (`mastery >= 7`) режим фиксирован: `Typing First Letters`
- Пайплайн усложнения:
  1. Click Chunks
  2. Click Words (hinted)
  3. Click Words (no hints)
  4. First Letters (hinted)
  5. First Letters Tap
  6. First Letters Typing
  7. Full Recall

## 4.4 Как проходят упражнения
- У каждого режима локальная механика проверки (порядок слов/букв, сброс при ошибке, подсказка).
- После успешного завершения режима показывается self-rating:
  - `0` — забыл
  - `1` — сложно
  - `2` — нормально
  - `3` — отлично

## 4.5 Обновление прогресса по рейтингу
- `MASTERY_DELTA_BY_RATING`:
  - `0 -> -1`
  - `1 -> 0`
  - `2 -> +1`
  - `3 -> +2`
- Правила:
  - В `LEARNING` mastery клампится в диапазон `1..7`.
  - В review (по сути после graduation) raw mastery может быть >= 0.
- `repetitions` растет только в review и только если rating != 0.
- Переходы после обновления:
  - если repetitions >= 5 -> `MASTERED`
  - иначе если mastery >= 7 -> `REVIEW`
  - иначе -> `LEARNING`

## 4.6 Расчет следующего повторения (`nextReviewAt`)
- Базовый интервал зависит от стадии mastery:
  - 0: 10 минут
  - 1: 1 час
  - 2: 6 часов
  - 3..7: 3 дня
- Множитель зависит от score (score берется из rating):
  - score >= 92: `1.25`
  - score >= 80: `1.0`
  - score >= 65: `0.75`
  - иначе: `0.5`

## 4.7 Сохранение на сервер
- Патч отправляется в `/api/users/{telegramId}/verses/{externalVerseId}`:
  - `masteryLevel`
  - `repetitions` (опционально)
  - `lastReviewedAt`
  - `nextReviewAt`
  - `lastTrainingModeId`
  - `status` (базовый: обычно `LEARNING`, либо `STOPPED`/`MY`)
- Серверные ограничения:
  - Для `LEARNING` mastery клампится в `1..7`
  - Нельзя менять repetitions, пока mastery < 7
  - Если у review-стиха `nextReviewAt` еще в будущем, сервер игнорирует изменение repetitions

## 5. Daily goal: когда прогресс считается выполненным

- События прогресса приходят из gallery/training.
- Для daily goal действуют более строгие правила:
  - Этап `learning` засчитывается только когда стих дошел до `mastery >= 7` (порог review).
  - Этап `review` засчитывается только если завершен режим `Typing First Letters` (mode id этого режима).
- После выполнения обеих фаз (или после learning, если review-этап пропущен) цель дня отмечается завершенной.

## 6. Движок тренировки унифицирован

- В runtime приложения оставлен один движок тренировки: `VerseGallery` + `useTrainingFlow`.
- Ветка запуска через `TrainingSession` удалена из `App.tsx`:
  - больше нет отдельного training-overlay и второго entrypoint,
  - нет дублирующего набора state/handlers для отдельной сессии.
- Практический эффект:
  - один источник истины для выбора режима, оценки, сохранения и daily goal-событий,
  - меньше риска расхождений в правилах прогресса и status-переходах.

## 7. Краткая формула «как стих становится выученным»

1. Стих переводится в `LEARNING`.
2. Пользователь проходит режимы, повышая mastery до `7`.
3. После достижения `7` стих становится `REVIEW`.
4. На review-повторах увеличивается `repetitions`.
5. Когда `repetitions >= 5`, display-статус становится `MASTERED`.

## Что можно исправить?

1. [Сделано] Унифицирован движок тренировки: в runtime оставлен `VerseGallery`, дублирующий поток `TrainingSession` убран из `App.tsx`.
2. Синхронизировать due-логику во всех местах:
   - сейчас readiness считает только due-review, а часть подборок/планов может брать review без проверки due.
3. Пересмотреть review-механику:
   - сейчас review фактически фиксирован на одном режиме (`Typing First Letters`).
   - лучше сделать ротацию review-режимов (например: First Letters -> Full Recall -> Speed Recall), чтобы снизить эффект «натаскивания на один паттерн».
4. Перейти на более формальную spaced repetition модель (например, SM-2/FSRS-подобную):
   - использовать не только self-rating, но и объективные метрики (ошибки, время, подсказки), чтобы интервалы были точнее.
5. Добавить режимы заучивания:
   - `Cloze` (пропуски слов в тексте),
   - `Reference Recall` (по тексту ввести ссылку на стих),
   - `Reverse Recall` (по ссылке воспроизвести текст без подсказок),
   - `Interleaving` (чередование 3-5 стихов в одной сессии),
   - `Audio Recite` (проговаривание с проверкой через STT).
6. Убрать неиспользуемые/устаревшие куски (или явно пометить как experimental), чтобы упростить поддержку и снизить риск регрессий.
