# Анализ логики заучивания стихов в Bible Memory App

## Профессиональная оценка системы запоминания

---

## 1. Архитектура жизненного цикла стиха (Verse Flow)

### 1.1 Фазы прохождения стиха

```
CATALOG → QUEUE → LEARNING → REVIEW → MASTERED
```

| Фаза | Код | Назначение |
|------|-----|------------|
| **Каталог** | `CATALOG` | Стих доступен в библиотеке, но не добавлен пользователю |
| **Очередь** | `QUEUE` | Добавлен в очередь на изучение (ограничение активных слотов) |
| **Изучение** | `LEARNING` | Активное заучивание (уровни мастерства 0-7) |
| **Повторение** | `REVIEW_DUE` / `REVIEW_WAITING` | Интервальные повторения по расписанию |
| **Заучено** | `MASTERED` | Стих полностью освоен (7+ повторений) |

### 1.2 Система пауз (Pause States)

Каждая активная фаза имеет приостановленный аналог:
- `PAUSED_LEARNING` — изучение приостановлено
- `PAUSED_REVIEW` — повторения приостановлены  
- `PAUSED_MASTERED` — архивированный освоенный стих

---

## 2. Двухуровневая модель прогресса

### 2.1 Мастерство (Mastery Level) — 0..7

| Уровень | Режим тренировки | Описание |
|---------|------------------|----------|
| 0 | ClickChunks | Крупные блоки текста на выбор |
| 1 | ClickWordsHinted | Слова с подсказками |
| 2 | ClickWordsNoHints | Слова без подсказок |
| 3 | FirstLettersWithWordHints | Первые буквы + подсказки слов |
| 4 | FirstLettersTapNoHints | Первые буквы, нажатие без подсказок |
| 5 | FirstLettersTyping | Набор по первым буквам |
| 6 | FullRecall | Полное воспроизведение |
| 7 | FullRecall | Выход в фазу повторений |

### 2.2 Повторения (Repetitions) — 0..7

После достижения мастерства 7 начинается фаза **spaced repetition** (интервальных повторений):

| Повтор | Интервал | Режим тренировки |
|--------|----------|------------------|
| 0 | 1 день | FirstLettersTyping |
| 1 | 3 дня | FullRecall |
| 2 | 7 дней | VoiceRecall |
| 3 | 14 дней | FullRecall |
| 4 | 30 дней | VoiceRecall |
| 5 | 60 дней | FullRecall |
| 6 | 90 дней | FullRecall |
| 7+ | 180 дней | Поддерживающий повтор |

---

## 3. Принципы вычисления прогресса

### 3.1 Единая шкала прогресса

```typescript
totalCompleted = min(masteryLevel + repetitions, 14)  // 7 + 7
totalRemaining = (7 - min(mastery, 7)) + (7 - min(repetitions, 7))
progressPercent = round(totalCompleted / 14 * 100)
```

**Ключевой инсайт**: система использует **линейную агрегацию** двух метрик в одну шкалу 0-14, где:
- Первые 7 единиц — фаза изучения (mastery)
- Следующие 7 единиц — фаза повторений (repetitions)

### 3.2 Прогрессия режимов обучения

```
Progress Order: [1,2,3,4,5,6,7] // ClickChunks → FullRecall
```

**Логика сдвига режима**:
- `rating = -1` ("забыл") → понижение режима (-1)
- `rating = 0` ("сложно") → сохранение текущего режима
- `rating = 1` ("далее") → повышение режима (+1)

---

## 4. Система интервальных повторений (Spaced Repetition)

### 4.1 Формула вычисления следующего повторения

**Для фазы LEARNING**:
```typescript
delayMs = baseDelay(stage) * scoreMultiplier(score)
where score ∈ {35, 60, 84} for ratings {-1, 0, 1}
```

**Базовые задержки по уровням**:
| Stage | Задержка |
|-------|----------|
| 0 | 10 минут |
| 1 | 1 час |
| 2 | 6 часов |
| 3 | 1 день |
| 4 | 2 дня |
| 5 | 3 дня |
| 6 | 5 дней |
| 7 | 7 дней |

**Мультипликаторы оценки**:
| Score | Мультипликатор |
|-------|----------------|
| ≥92 | 1.25x |
| 80-91 | 1.0x |
| 65-79 | 0.75x |
| <65 | 0.5x |

### 4.2 Поведение при неудачном повторении

**Ранняя фаза повторений (repetitions 0-3)**:
- Первый провал: retry через 6 часов, накопление "strike"
- Второй провал подряд: откат repetitions на 2 единицы

**Поздняя фаза повторений (repetitions 4+)**:
- Провал: retry через 6 часов **без штрафа** к repetitions
- Логика: "пользователь доказал долгосрочное запоминание — жёсткий откат демотивирует"

## 5. Система подсказок (Hint Engine)

### 5.1 Типы подсказок

| Тип | Описание | Влияние на оценку |
|-----|----------|-------------------|
| `next_word` | Показ следующего ожидаемого слова | Ограничение maxRating до 0 |
| `full_text_preview` | Показ всего стиха на N секунд | Ограничение maxRating до 0 |
| `full_text` | Полный показ с блокировкой ввода | Статус "surrendered" |

### 5.2 Лимиты использования подсказок

| Сложность стиха | Max `next_word` подсказок |
|-----------------|---------------------------|
| EASY | 1 |
| MEDIUM | 2 |
| HARD | 3 |
| EXPERT | 3 |

**Важное правило**: использование ЛЮБОЙ подсказки ограничивает максимальную оценку до 0 ("сложно"), что означает:
- Нельзя получить "далее" (rating=1) после подсказки
- Нельзя понизить режим при rating=-1 в review-фазе

## 6. Механизмы оценивания

### 6.1 Доступные оценки по фазам

**LEARNING фаза**:
- `-1` ("забыл") — понижение mastery level
- `0` ("сложно") — сохранение текущего уровня
- `1` ("далее") — повышение mastery level

**REVIEW фаза**:
- `0` ("сложно") — провал, retry через 6 часов
- `1` ("далее") — успех, увеличение repetitions

**Без подсказок** (`assisted=false`):
- Полный набор оценок доступен

**С подсказками** (`assisted=true`):
- Только `0` ("сложно") — принудительный повтор

### 6.2 Дельты изменения мастерства

```typescript
RATING_MASTERY_DELTAS = { [-1]: -1, [0]: 0, [1]: 1 }
```

## 7. Критические правила системы

### 7.1 Условие выхода в Review

```typescript
// Выпуск в review ТОЛЬКО при:
graduatesToReview = (masteryLevel >= 7) && (trainingModeId === TrainingModeId.FullRecall)

// Если mastery=7 но режим ≠ FullRecall:
nextRawMasteryLevel = 6  // Сброс на уровень назад
graduatesToReview = false
```

**Профессиональная оценка**: это **жёсткий фильтр качества** — пользователь должен пройти полное воспроизведение для перехода к интервальным повторениям.

### 7.2 Late Stage Review (repetitions ≥ 4)

- Нет кнопки "Забыл" (только "Сложно" / "Далее")
- Нет подсказок
- Мягкие штрафы при провале (только retry, без отката repetitions)

**Обоснование**: пользователь доказал долгосрочное запоминание через 4+ успешных повторения.

## 8. Поток состояния тренировочной сессии

```
active → awaiting_rating → finalized
   ↓           ↓
surrendered  abandoned
```

| Состояние | Описание |
|-----------|----------|
| `active` | Пользователь в процессе ввода |
| `awaiting_rating` | Упражнение завершено, ожидается оценка |
| `finalized` | Оценка выставлена, сессия завершена |
| `surrendered` | Пользователь сдался (показал полный текст) |
| `abandoned` | Сессия прервана без сохранения |

## 10. Профессиональные выводы

### 10.1 Сильные стороны системы

1. **Чёткая градуировка сложности** — 8 режимов с прогрессивным усложнением
2. **Адаптивная система подсказок** — лимитированные подсказки с ценой в оценке
3. **Защита от "прокачки"** — невозможно выйти в review без FullRecall
4. **Умная обработка провалов** — различная логика для ранних/поздних повторений
5. **Единая метрика прогресса** — простая для понимания шкала 0-100%
6. **Spaced repetition с адаптивными интервалами** — мультипликаторы по качеству ответа

### 10.2 Нюансы реализации

1. **Hint penalty** — любая подсказка = "сложно", даже если пользователь знал ответ
2. **Mastery clamping** — в learning-фазе mastery ограничен [1..7], в review [0..∞)
3. **Review rotation** — фиксированная последовательность режимов, не адаптивная
4. **Late stage threshold** — на repetitions=4 меняется вся философия обработки ошибок

### 10.3 Паттерны запоминания

Система реализует **трёхкомпонентную модель** запоминания:
1. **Конструирование** (learning, modes 1-6) — построение памятного следа
2. **Укрепление** (review, reps 0-6) — интервальное укрепление
3. **Поддержание** (mastered, maintenance) — разовые повторения каждые 180 дней
